## Compliance scope

`a2ui-svelte` is **100% compatible** with the A2UI v0.8 specification on
its default surface wire:

- All 16 standard catalog components, with their spec-defined props.
- All four server‚Üíclient message kinds ‚Äî `surfaceUpdate`,
  `beginRendering`, `dataModelUpdate`, `deleteSurface`.
- Both client‚Üíserver events ‚Äî `userAction` (with the spec-mandated
  `{ name, surfaceId, sourceComponentId, timestamp, context }` shape;
  `context: {}` when the source component declares none) and `error`.
- The two spec-canonical generic tools `click_button({element_id})`
  and `update_text_field({element_id, value})`.
- The catalog-selection handshake (client capabilities under
  `a2uiClientCapabilities`; standard-catalog URI default).
- The A2A `DataPart` envelope (`mimeType: "application/json+a2ui"`,
  `X-A2A-Extensions` header).

A small number of pre-spec behaviours (surface-change polling, batched
click/update tools, richer tool-result envelope, XML-tagged-text
`userAction` for voice live-APIs) ship behind a per-surface flag and
emit their data under `extensions: { 'a2ui-svelte': ‚Ä¶ }`. Spec-strict
consumers drop the namespace and still see exactly what v0.8 promises.
See the [extensions guide](../guides/extensions.md) for the full
boundary.

# A2UI Protocol <span class="version-badge stable">v0.8 — Stable</span>
>
> Version 0.8 is the current stable release, recommended for production use.

> Last reviewed: 2026-05-10. Maintained at `a2ui-svelte/docs/specification/`.
> The Svelte runtime in this package implements this version of the
> protocol.

## Compliance scope

The JSON `a2ui-svelte` emits and accepts **conforms to the A2UI v0.8
schemas** on its default surface wire. Two caveats matter before you
rely on it (full story in
[What `a2ui-svelte` is](../guides/a2ui-compatibility.md)):

- The best-supported pattern, `<StaticSurface>`, is *not* classic A2UI —
  you own the UI and the agent reads/drives it. `<DynamicSurface>` (the
  agent-renders-the-UI model) is experimental.
- Connecting a 3rd-party agent over A2A requires you to implement the
  transport yourself — only the interface, envelope helpers, and the
  `<A2ASurface>` adapter ship.

What conforms on the wire:

- All 16 standard catalog components, with their spec-defined props.
- All four server→client message kinds — `surfaceUpdate`,
  `beginRendering`, `dataModelUpdate`, `deleteSurface`.
- Both client→server events — `userAction` (with the spec-mandated
  `{ name, surfaceId, sourceComponentId, timestamp, context }` shape;
  `context: {}` when the source component declares none) and `error`.
- The two spec-canonical generic tools `click_button({element_id})`
  and `update_text_field({element_id, value})`.
- The catalog-selection handshake (client capabilities under
  `a2uiClientCapabilities`; standard-catalog URI default).
- The A2A `DataPart` envelope (`mimeType: "application/json+a2ui"`,
  `X-A2A-Extensions` header).

A small number of pre-spec behaviours (surface-change polling, batched
click/update tools, richer tool-result envelope, XML-tagged-text
`userAction` for voice live-APIs) ship behind a per-surface flag and
emit their data under `extensions: { 'a2ui-svelte': … }`. Spec-strict
consumers drop the namespace and still see exactly what v0.8 promises.
See the [extensions guide](../guides/extensions.md) for the full
boundary.
